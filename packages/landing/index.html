How it works

One environment variable.
Complete cost control.

Point your agents at the SteadIO proxy instead of OpenAI or Anthropic directly. That's it.

1

Start the proxy

Run docker compose up -d to start the SteadIO proxy, cost engine, and dashboard. Zero config needed.

2

Point your agents at it

Set OPENAI_BASE_URL=http://localhost:3001/openai. No SDK changes, no wrappers. Works with Python, TypeScript, Go.

3

Set budget caps

Define per-agent or per-team caps. When an agent hits its limit, SteadIO returns HTTP 402 and the agent stops. No bill.

4

Watch the dashboard

Real-time cost attribution per agent and team. Drill into any agent to see exactly which model and how many tokens.

Quick start

Up in 60 seconds

Three commands to full budget enforcement. No API keys needed for the demo.

# 1. Clone and start the stack
git clone https://github.com/steadioai/steadio
cd steadio && make demo

# 2. Point your agent at the proxy (no code changes)
export OPENAI_BASE_URL=http://localhost:3001/openai
export ANTHROPIC_BASE_URL=http://localhost:3001/anthropic

# 3. Set a $10/day budget cap for your agent
curl -X POST http://localhost:3002/api/budgets \
  -d '{"scope":"agent","scopeId":"my-agent","period":"daily","capUsd":10,"enforcementMode":"kill"}'

# Agent hits $10.01 → proxy returns 402, agent stops
{"error":"budget_exceeded","agent_id":"my-agent","cap_amount":10,"current_spend":10.01}
Features

Everything you need to run
agents safely in production

Built for teams running autonomous AI agents at scale.

🛑

Hard budget enforcement

Returns HTTP 402 the moment an agent hits its cap. Not alerts — actual enforcement. Your downstream LLM bill stops.

⚡

Runaway detection

Detects velocity spikes and repeated identical prompts before a budget cap is even hit. Opens a circuit breaker automatically.

📊

Per-agent attribution

Every token tagged by agent ID and team. See exactly which agent caused a cost spike — down to the individual request.

🔌

Language-agnostic

One environment variable. Works with Python, TypeScript, Go, or anything that makes HTTP calls. No SDK wrappers to maintain.

🌊

Streaming supported

Full SSE streaming passthrough with token counting mid-stream. Budget enforcement fires at the right moment, not end-of-response.

🏠

Self-hosted

Your data stays in your infrastructure. Runs on Docker Compose. PostgreSQL for storage, Redis for real-time enforcement.
